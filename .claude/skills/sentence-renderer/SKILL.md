---
name: sentence-renderer
description: >
  Use this skill when the user writes "render sentences".
  Scans data/sentences/ for JSON files generated by the sentence-builder skill,
  checks data/sentences-html/ for already-rendered files (by matching filename stems),
  and generates a styled HTML page for each unrendered JSON.
  Saves output to data/sentences-html/ using the same filename stem as the source JSON.
---

## Allowed Tools
* Bash: to list directory contents and create directories
* Read: to read each unrendered JSON file
* Write: to save the generated HTML files

---

## Step 1 — Discover Files

Use Bash to list existing files:

```bash
ls /path/to/data/sentences/*.json 2>/dev/null
ls /path/to/data/sentences-html/*.html 2>/dev/null
```

Use paths relative to the project root:
- JSON source: `data/sentences/`
- HTML output: `data/sentences-html/`

Build two sets:
- **json_stems**: the filename stem (basename without extension) of each `.json` file in `data/sentences/`, excluding `.gitkeep` and any non-JSON files
- **html_stems**: the filename stem of each `.html` file in `data/sentences-html/`

A JSON file is **unrendered** if its stem does NOT appear in `html_stems`.

If all files are already rendered (or there are no JSON files), report:

> "All sentence files are already up to date — nothing to render."

and stop.

---

## Step 2 — Read Each Unrendered JSON

For each unrendered JSON file, use the Read tool to load it from `data/sentences/{stem}.json`.

The JSON schema is:

```json
{
  "word": "<word as typed>",
  "base_form": "<dictionary form>",
  "word_type": "<noun|verb|adjective|adverb|other>",
  "generated_at": "<ISO 8601 datetime>",
  "sentences": [
    {
      "id": 1,
      "german": "...",
      "english": "...",
      "level": "simple|medium|complex",
      "grammar_note": "..."
    }
  ]
}
```

Group the sentences by level:
- **Simple**: `level == "simple"` (IDs 1–3)
- **Medium**: `level == "medium"` (IDs 4–6)
- **Complex**: `level == "complex"` (IDs 7–10)

## Step 3 — Generate the HTML

For each unrendered JSON, produce a self-contained, styled HTML file (inline CSS only — no external dependencies).

### Page title
`Sentences: {word} ({word_type}) — A1`

### Structure

```
<html lang="en">
  <head>  — charset, viewport, title, inline CSS
  <body>
    <div class="container">
      <header>   — word, word type pill, generated date
      <nav>      — anchor links: Simple | Medium | Complex
      <section id="simple">   — simple sentence cards
      <section id="medium">   — medium sentence cards
      <section id="complex">  — complex sentence cards
      <footer>   — "Generated by German Learning Assistant · A1 · {date}"
    </div>
  </body>
</html>
```

### Header content
- `<h1>` — `Sentences: {word}`
- A word-type pill badge (same style as level badges, use a neutral color: `#37474f`)
- `<p class="meta">` — `Generated {YYYY-MM-DD} · Level: A1`

### Nav
Three anchor links:
- `📗 Simple (3)`  → `#simple`
- `🟠 Medium (3)`  → `#medium`
- `🔴 Complex (4)` → `#complex`

### Sentence sections
Each section has:
- `<h2>` with a level badge and count, e.g. `Simple — 3 sentences`
- One `.sentence-card` per sentence in that group

Each `.sentence-card` contains:
1. A header row: sentence number + level badge
2. The German sentence — large, bold, indigo (`#1a237e`)
3. The English translation — prefixed with `→`, muted gray (`#555`)
4. The grammar note — prefixed with `📝`, italic, purple-ish (`#5c6bc0`)

## Step 4 — Save the HTML File

Use Bash to ensure the output directory exists:

```bash
mkdir -p data/sentences-html
```

Use the Write tool to save each HTML file to:
- `data/sentences-html/{stem}.html`

Where `{stem}` is the exact filename stem of the source JSON (e.g., if the JSON is `essen_2026-03-04_000000.json`, the HTML is `essen_2026-03-04_000000.html`).

## Step 5 — Regenerate index.html

After saving all newly rendered HTML files, regenerate `index.html` at the project root so new files appear in the browser index.

### How to rebuild index.html

1. Use Bash to list all HTML files in both output directories:
   ```bash
   ls data/grammar-html/*.html 2>/dev/null
   ls data/sentences-html/*.html 2>/dev/null
   ```

2. For each filename, extract:
   - **label**: strip the datetime suffix (`_YYYY-MM-DD_HHMMSS`) from the stem, replace remaining underscores with spaces; capitalize the first letter of the result. Use your own knowledge of the word to improve capitalization where appropriate (e.g., `die_kunst` → "die Kunst", `perfekt_past_tense` → "Perfekt Past Tense").
   - **date**: the `YYYY-MM-DD` portion embedded in the filename (second-to-last underscore segment, format `YYYY-MM-DD`).
   - **href**: the relative path from the project root, e.g. `data/sentences-html/{filename}`.

3. Sort each section by date descending (newest first).

4. Write `index.html` to the project root using the Write tool, following this structure exactly:
